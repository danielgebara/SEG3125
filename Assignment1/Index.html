How I work

I don't have a formal design process yet. This is just how I've been approaching things so far.

Figure out what the thing is supposed to do

Before writing any code I try to think about who is actually using it and what they need. SEG3125 has been pushing me to be more deliberate about this instead of jumping straight into building.

Sketch it out first

Even a rough drawing on paper helps. I didn't used to do this but it saves a lot of time compared to redesigning things mid-build.

Bootstrap for structure, CSS for the rest

Bootstrap gets the layout working, then I go in with CSS to make it look the way I want. I use W3Schools and Net Ninja as references.

Apply what we learn in class

Visual hierarchy, contrast, balance, scale. Concepts from SEG3125 and the NN/g videos. I'm still getting used to thinking this way but it's starting to click.
